Задача на массивы
1. Создать функцию генерации массива произвольной вложенности (глубина от 1 до 5, например, [123, text, [123, […]] и так далее) с произвольным количеством элементов (от 1 до 10) произвольного типа (boolean, float, integer, array, string). Все действия разбить по функциям. Если тип элемента массива – array, то необходимо его генерировать таким же образом, как общий массив. Функция генерации массива должна отдавать случайный массив при вызове сразу. 
2. В полученном массиве найти и вывести на экран (включая все вложенные массивы):
а) самый большой числовой элемент
б) строку с самым большим количеством заглавных букв
в) массив, содержащий наибольшее количество целочисленных элементов
б) массив, содержащий наименьшее число элементов
3. Во всех массивах, все строковые и логические элементы заменить случайными числами (int и float, через один), и отсортировать по значениям в обратном порядке. Вывести результат на экран.

Тестовое задание для fullstack разработчика
 Реализовать мини-интернет-магазин, включающий 2 экрана – каталог товаров и корзину.
Каталог должен содержать минимум 5 товаров. В карточке товара должно присутствовать следующее:
Свойства товара
Наименование
Изображение
Краткое описание
Цена за единицу
Артикул

У каждого товара должна быть кнопка "Добавить в корзину" и поле ввода количества (которое добавляем в корзину).
 Количество должно быть больше 0 и меньше 1000
 По кнопке "добавить в корзину" добавляем товар в корзину, и выводим сообщение "Товар успешно добавлен"
 Если мы добавляем тот же самый товар, то их количество в корзине суммируется.
 На странице есть кнопка "Показать корзину", по нажатию на которую появляется всплывающее окно (модальное), которое отображает список товаров в корзине.

Формат корзины
Изображение
Артикул
Наименование
Цена за единицу
Кол-во
Общая стоимость позиции
Также в корзине отображается общая стоимость и общее кол-во товаров, а также имеется кнопка «оформить заказ», которая очищает корзину и добавляет запись о заказе и его составе в БД.

Для фронтенда необходима адаптивная верстка с применением технологий HTML, CSS, JS. Допустимо использование библиотеки React. Можно использовать CSS фреймворки (Bootstrap и тд), можно использовать redux, redux/toolkit, react context, typescript и т.д.
Бэкенд необходимо реализовать на чистом PHP с использованием PDO, принципов ООП, а также написание кода с применением стандартов PSR.
В качестве БД использовать MySQL.
 
Готовый результат необходимо опубликовать в публичном репозитории Gitlab или Github. Дамп базы приложить в формате .sql. В файле readme.md написать небольшую документацию по разворачиванию проекта на сервере (указать версии php и mysql, необходимые консольные команды, если они использовались). Так же необходимо разместить рабочий проект в глобальной сети.
